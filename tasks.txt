1. все правки должны быть сделаны в отдельной ветке, и результатом должен быть pull request в ветку master
2. ввести пользователей в твой проект через "стандартные" механизмы фреймворка.
3. должен быть root-пользователь, который будет иметь возможность редактировать через веб-интерфейс добавлять/редактировать/удалять пользователей, задавать и менять им пароли
4. неавторизованным пользователям - не показывать твой проект, только страницу входа
5. авторизованным пользователям - показывать весь текущий функционал, плюс кнопку Выход (с соответствующим функционалом)
6. сделать возможность пользователям лайкать встречи (и снимать лайк по повторному нажатию).
7. около встречи показывать количество лайков и логины лайкнувших пользователей
